Task 1: User & Permission Misconfigurations

1. Executive Summary

This PoC demonstrates how incorrect permissions on sensitive system files (e.g., /etc/shadow) can allow low-privileged users to access critical information. The task involves creating users, misconfiguring file permissions, exploiting the misconfiguration, and then mitigating the issue by restoring proper permissions and ownership.

2. Setup

Create Users

sudo useradd jesse
sudo passwd jesse
sudo useradd pinkman
sudo passwd pinkman

Assign Incorrect Permissions

sudo chmod 777 /etc/shadow

3. Exploitation

su jesse
cat /etc/shadow
root:1:20154:0:99999:7::: daemon:*:20154:0:99999:7::: bin:*:20154:0:99999:7:::

4. Mitigation

sudo chmod 640 /etc/shadow
sudo chown root:shadow /etc/shadow
ls -l /etc/shadow
-rw-r----- 1 root shadow 1628 Mar 10 09:44 /etc/shadow

Recommendations:

  • Regularly audit file permissions on critical system files
  • Use tools like auditd or tripwire to monitor changes to sensitive files
  • Educate system administrators about proper file permissions

Task 2: Remote Access & SSH Hardening

1. Executive Summary

This PoC demonstrates the risks associated with insecure SSH configurations, such as allowing root login and password-based authentication. The task involves enabling SSH, exploiting weak configurations through a brute-force attack, and mitigating the risks by hardening the SSH configuration.

2. Setup

sudo systemctl start ssh
sudo systemctl enable ssh

Edit /etc/ssh/sshd_config:

PermitRootLogin yes
PasswordAuthentication yes

3. Exploitation

hydra -l jude -P wordlist.txt ssh://localhost

4. Mitigation

Edit /etc/ssh/sshd_config:

PermitRootLogin no
PasswordAuthentication no
sudo systemctl restart ssh

Key-Based Authentication

ssh-keygen -t rsa -b 4096
ssh-copy-id jude@localhost

Recommendations:

  • Regularly update SSH configurations to follow security best practices
  • Use tools like fail2ban to block repeated failed login attempts
  • Monitor SSH logs for suspicious activity

Task 3: Firewall & Network Security

1. Executive Summary

This PoC demonstrates the risks of having a web server exposed without proper firewall rules. The task involves setting up a basic web server, scanning for open ports, and mitigating risks by configuring a firewall to restrict access.

2. Setup

sudo systemctl start apache2
sudo systemctl enable apache2
sudo ufw disable

3. Exploitation

nmap 127.0.1.1
PORT STATE SERVICE 22/tcp open ssh 80/tcp open http
echo -en "GET / HTTP/1.1\r\nHost: 127.0.0.1\r\nConnection: close\r\n\r\n" | nc 127.0.0.1 80

4. Mitigation

sudo ufw enable
sudo ufw allow ssh
sudo ufw allow http
sudo ufw status verbose

IPTables Rules

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -j DROP

Recommendations:

  • Regularly review and update firewall rules
  • Use tools like fail2ban to block repeated failed login attempts
  • Monitor network traffic for suspicious activity

Task 4: SUID Misconfigurations & Privilege Escalation

1. Executive Summary

This PoC demonstrates the risks associated with SUID (Set User ID) misconfigurations, which can allow low-privileged users to escalate their privileges to root. The task involves setting up SUID misconfigurations, exploiting them to gain root access, and mitigating the risks by removing unnecessary SUID permissions.

2. Setup

Set SUID Bit on /bin/bash

sudo chmod u+s /bin/bash
ls -l /bin/bash
rwsr-xr-x 1 root root 1302512 Feb 16 03:57 /bin/bash

Create Script with SUID Permissions

echo "echo 'Running as root!'" > script.sh
sudo chmod 4755 script.sh
sudo chown root.root script.sh

3. Exploitation

find / -perm -4000 2>/dev/null
/home/jude/script.sh /usr/bin/su /usr/bin/mount
/bin/bash -p
bash-5.2# (root shell achieved)

4. Mitigation

sudo chmod -s /bin/bash
sudo chmod 750 script.sh
sudo chown root:root script.sh

Recommendations:

  • Regularly audit SUID/SGID files on the system
  • Remove SUID/SGID permissions from files that don't require them
  • Use tools like lynis or tripwire to monitor file permissions

Task 5: Automated Security Auditing & Scripting

1. Executive Summary

This PoC demonstrates how a Bash script can automate security auditing tasks, including monitoring login attempts, running services, and disk usage. The script helps identify misconfigurations and provides mitigation steps like cron automation and alerts.

2. Security Audit Script (security_audit.sh)

#!/bin/bash REPORT_FILE="security_audit_report.txt" echo "Security Audit Report - $(date)" > $REPORT_FILE # Check login attempts echo -e "\n=> User Login Attempts ==" >> $REPORT_FILE last -n 10 >> $REPORT_FILE grep "sshd" /var/log/auth.log | tail -n 10 >> $REPORT_FILE # Check running services echo -e "\n=> Running Services ==" >> $REPORT_FILE systemctl list-units --type-service --state-running >> $REPORT_FILE # Check disk usage echo -e "\n=> Disk Usage ==" >> $REPORT_FILE df -h >> $REPORT_FILE echo -e "\nAudit completed. Report saved to $REPORT_FILE"

3. Execution

bash security_audit.sh
Audit completed. Report saved to security_audit_report.txt

4. Mitigation

Cron Automation

0 3 * * * /path/to/security_audit.sh

Stop Unnecessary Services

sudo systemctl stop apache2 ssh

Enable fail2ban

sudo systemctl start fail2ban
sudo systemctl enable fail2ban

Recommendations:

  • Modify the script to send reports via email
  • Combine with tools like lynis for deeper scans
  • Allow only essential services

Task 6: Log Analysis & Intrusion Detection

1. Executive Summary

This PoC demonstrates the importance of log analysis and intrusion detection in identifying and mitigating brute-force attacks and unauthorized access attempts. The task involves enabling system logging, simulating failed SSH login attempts, analyzing logs, and implementing mitigation measures.

2. Setup

sudo systemctl start rsyslog
sudo systemctl enable rsyslog

Simulate Failed Logins

ssh user@localhost

3. Log Analysis

grep "Failed password" /var/log/auth.log
grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -nr

4. Mitigation

Configure fail2ban

sudo nano /etc/fail2ban/jail.local
[sshd] enabled = true maxretry = 3 bantime = 3600
sudo systemctl restart fail2ban

Log Monitoring

sudo logwatch --detail high --mailto jude@gmail.com --range today

Recommendations:

  • Regularly review logs for suspicious activity
  • Use tools like fail2ban to automatically block malicious IPs
  • Set up automated log monitoring and reporting
